          视联网会议感知系统操作手册

          VisionVera 2022-07-15 会议感知系统

          # 产品介绍

          视联网会议感知V1.3.0是一款可以感知到在视联网内所有正在进行中的会议，并快速感知到每场会议参会终端的状态变化，根据参会终端状态变化，协助运维人员快速排查故障原因，恢复参会终端正常参会的一个感知系统。

          # 登陆方式

          谷歌浏览器登陆：http://192.168.10.165:8088/

          输入用户名和密码：

          初始用户名为：admin

          初始密码为：123456

          点击登陆即可！如图所示：

          # 视联网会议感知“首页”功能

          登录该系统后将进入视联网会议感知“首页”页面，该页面是对所有当前进行中会议的总览。如图1-1所示

          图 1-1

          # 会议进度

          根据图1-1会议进度列可以清晰的看到每场会议的时间进度，是否超出预约时间，以及何时出现参会方点位故障等信息，根据以上信息，如果有必要会议，运维人员可以直观的看到控会人，通知控会人相关信息；

          # 会议进度条颜色说明

          底色为绿色：代表该场会议参会终端均为正常参会；如图1-2

          图 1-2

          底色为红色：代表该场会议的参会终端在某段时间有故障终端，鼠标移动至进度条可以看到故障发生的具体时间；如图1-3

          图 1-3

          进度条出现黄色线条：代表预约会议的终止时间，黄色线条后边是超出预约时间的进度；如图1-2

          # 会议名称操作说明

          查看某场会议的点位故障情况或者点位详情，可以直接点击该页面“会议名称”跳转到进行中页面，如图1-1会议名称列

          # 顶置功能

          考虑到当前进行的会议数量可能较多，而我们想要关注的会议又比较靠后，不便于观察想要关注的会议，所以设置了顶置功能；如图1-4，鼠标悬疑到序号，当出现图标时单机即可顶置该会议；取消顶置：仅针对已顶置会议，鼠标点击序号前图标即可；

          图 1-4

          # 视联网会议感知“进行中”页面

          该页面展示的是所有正在进行中的会议，以及每场会议的所有参会点位的故障情况；如图1-5所示

          图 1-5

          # 会议列表展示说明

          如图1-6进行中页面的左侧为会议列表，该列表所展示的是所有正在进行的会议名称、会议开始时间、参会终端总数、参会终端故障数、以及会议等级

          图 1-6

          # 点击会议列表中会议名称操作说明

          点击会议列表中的会议名称可以在右侧点位详情页面展示该会议的所有参会终端，以及参会终端所在的2K名称、2K转发流量，自治名称

          # 点击会议列表中参会终端故障数操作说明

          点击会议列表参会终端故障数，可以查看所点中会议的故障终端的具体明细

          # 会议列表中的会议开始时间说明

          会议列表中的会议开始时间为实际会议开始时间，非预约开始时间

          # 点位明细页面说明

          该页面展示的是所选中会议的所有参会终端、终端所在2K、终端所造自治的故障感知情况，以及2K和参会终端的收发量情况，具体标识含义如图1-7

          图 1-7

          # 故障标识说明

          2K 故障标识： 故障 正常 自治故障标识：故障 正常

          参会终端：底色为红色故障 底色为蓝色正常

          # 参会终端、终端所在2K、终端所在自治基本信息查找

          查找某个终端点位、终端所在2K、终端所在自治的基本信息，直接点击点位明细页面的点位框即可进入点位详情框，如图1-8

          图 1-8

          # 查看2K转发流量说明

          点击2K名称即可进入2K转发流量明细，该转发流量记录当前所有2K下在线终端的流量转发明细，如图1-9

          图 1-9

          # 终端状态变化页面说明

          点击图1-10圈中区域可以查看所选中会议的所有终端状态变化，包含“终端入会”、“终端退会”、“终端发言”、“终端故障变正常”、“终端正常变故障”，该功能可查询某个终端自入会后的所有状态变化，为终端发生故障分析提供可视化参考；该状态页面可根据终端名称进行搜索，方便排查某个指定终端状态变化，也可以根据状态类型进行筛选；

          图 1-10

          # 会议拓扑图说明

          （1）进入拓扑图操作说明

          在会议列表选中某场会议，单击“右键”，选择“查看拓扑图”即可进入拓扑图页面，如图1-11

          图 1-11

          （2）拓扑图说明

          拓扑图页面展示的是某场会议所有参会终端所在的自治、2K的连接关系，拓扑图中点位的不同颜色、不同位置代表不同的含义，具体含义如图1-12所示

          图 1-12

          # 转发关系

          （1）进入“转发关系”操作说明

          ​ 在会议列表选中某场会议，单击“右键”，选择“显示转发关系”即可进入2K转发关系页面，如图1-13

          图 1-13

          （2）“2K转发关系”页面说明

          查看会议转发关系，2K微云服务器之间存在转发关系，一旦某一台2K微云服务器出现故障，那么该2K微云服务器下所有参会点位都受到影响。2K转发关系可以看到会议中的业务流是如何走，通过了那些服务器和自治，以及所有参会2K的收发流量，方便运维人员对异常的定位。如图1-14

          图 1-14

          注：2K状态条为蓝色代表2K正常，状态条为红色代表2K故障

          # 主席终端物理关系

          （1）进入“物理关系”操作说明

          ​ 在会议列表选中某场会议，单击“右键”，选择“物理关系”即可进入物理关系页面，如图1-15

          图 1-15

          （2）“物理关系”页面说明

          会议进行时，经常出现网络繁忙的情况，一旦出现这种情况，参会点位将受到影响。如果主席终端出现网络繁忙，甚至将影响整个会议。主席终端物理关系图可以展示主席终端涉及到的物理链路上所有节点信息，包括交换机、2K、会管、网管、自治云等，如果某个节点发生异常，在物理关系图中可以体现，从而方便会议报障人员定位问题。如图1-16

          图 1-16

          注：点位出现红色标识时代表故障

          # 视联网会议感知“预约”页面

          会议预约页面展示的是本地区内预约的会议，点击页面“预约”即可进入，如图1-17

          图 1-17

          # 视联网会议感知“凝视”页面

          “凝视”页面展示的是所有所有正在进行中会议的参会终端、终端所在2K、以及终端所在自治的状态变化；对于异常的状态变化会在右边页面“告警日志”中告警，例如终端断网或者断电的情况；

          图 1-18
